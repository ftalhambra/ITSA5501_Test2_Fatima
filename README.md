☸️ Kubernetes Implementation
1️⃣ Namespace
A dedicated namespace named exam-system was created to isolate all Kubernetes resources used in this assessment.
File: k8s/namespace.yaml


2️⃣ PersistentVolumeClaim (PVC)
A PersistentVolumeClaim was created to support shared storage between containers.
PVC Specifications:

Name: exam-shared-pvc
Namespace: exam-system
Access Mode: ReadWriteOnce
Storage Request: 750Mi
File: k8s/pvc.yaml


3️⃣ Deployment (Multi‑Container Pod)
A Kubernetes Deployment named exam-app was configured with the following characteristics:

Namespace: exam-system
Replicas: 1 (initially)
Two containers in the same pod:
lighttpd-container (implemented using Alpine Linux due to local Docker image availability)
logger-container (BusyBox)


Shared PVC mounted in both containers
CPU and memory requests and limits defined for the main container

Key features demonstrated:
Multi‑container pod design
Shared storage usage
Resource management (requests & limits)
File: k8s/deployment.yaml

4️⃣ Scaling the Application
The deployment was scaled from 1 replica to 3 replicas using kubectl without editing the YAML file, as required by the test instructions.
Shellkubectl scale deployment exam-app --replicas=3 -n exam-systemShow more lines
The scaling was verified by confirming three running pods, each containing two running containers.

📊 Monitoring Implementation
1️⃣ Prometheus Configuration
Prometheus was configured for node‑level monitoring and self‑monitoring with the following settings:

Scrape interval: 45 seconds
Evaluation interval: 45 seconds
Node Exporter metrics (CPU, memory, load averages)
Prometheus self‑monitoring
Alert rules loaded from an external rule file

Files:

monitoring/prometheus-deployment.yml
monitoring/prometheus-service.yml
monitoring/custom_alert_rules.yml

Prometheus was accessed via kubectl port‑forward due to Docker Desktop networking limitations.

2️⃣ Alertmanager Configuration
Alertmanager was deployed to handle alert notifications generated by Prometheus.
Two alert rules were configured:

LowCPUUtilization
Triggered when CPU usage remains below 50% for more than 1 minute
LowMemoryUsage
Triggered when memory usage falls below 50% for more than 2 minutes

Files:
monitoring/alertmanager-deployment.yml
monitoring/alertmanager-service.yml

Alerts were verified through the Prometheus Alerts page.

📝 Notes & Assumptions
Docker Desktop was used as the local Kubernetes environment.
The original lighttpd:latest image could not be pulled due to Docker Hub access limitations; therefore, an equivalent alpine:latest image was used to demonstrate Kubernetes deployment, volume sharing, and scaling concepts.
This substitution does not affect the learning outcomes or assessment criteria, as the focus is on Kubernetes configuration and behavior rather than the web server implementation itself.
